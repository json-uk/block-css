1Objects #permalink

1.1Container #permalink

A container is any division of content designed to encapsulate sections of the page; e.g., the site header. The .Container class sets the max-width and (optionally) outer gutters.

Containers are always centered in the viewport and max-width set by the --site-width--max variable.

  • .Container
    Creates a new container context
  • .Container--withGutter
    Adds .5vr gutters above sm viewport and 1vr on the md viewport.

1.2Grid #permalink

The BlockCSS grid system is simple in nature, yet very powerful.

  • .Grid
    Creates the parent grid context.
  • .Grid-item
    CSS class for each immediate child grid column.

1.2.1Grid System Modifiers #permalink

Modifiers are special classes which extend the functionality of the grid system. Use them in conjunction with the .Grid and .Grid-item classes.

1.2.1.1Grid Alignment Modifiers #permalink

Aligns all grid cells by adjusting text-align and vertical-align properties.

  • .Grid--alignCenter
    horizontally centers .Grid-items.
  • .Grid--alignRight
    horizontally aligns .Grid-items to the right.
  • .Grid--alignLeft
    horizontally aligns .Grid-items to the right.
  • .Grid--alignMiddle
    vertically aligns .Grid-items to the midde of parent.
  • .Grid--alignBottom
    vertically aligns .Grid-items to the bottom of parent.

1.2.1.2Grid Item Alignment #permalink

  • .Grid-item--center
    Centers a single grid cell. All other .Grid-items remain the same.

1.2.1.3Grid Gutter Modifiers #permalink

This adds gutters to the .Grid-items.

  • .Grid--withGutter
    Adds left and right padding equal to --Grid-gutter variable.
  • .Grid--withGutterX2
    Adds twice as much left and right padding.

1.3Media #permalink

Used to create container for media such as: images and video. Immediate child node must have a class of .Media-object .

Example 
  <figure class="Media">
    <img class="Media-object" src="//dummyimage.com/600/545454/f5f5f5.jpg"/>
  </figure>

1.3.1Flexible Media #permalink

Buy adding a class of .Media--flex to the Media container and combined with .Media--aspectRatioXbyY classes creates a container that maintains aspect ratio by using padding.

The .Media-object is absolutely positioned within the box and stretched to the edges.

Examples
Default styling
.Media--flex.Media--ratio16by9
16:9 aspect ratio
.Media--flex.Media--ratio5by4
5:4 aspect ratio
.Media--flex.Media--ratio4by3
4:3 aspect ratio
.Media--flex.Media--ratio3by2
3:2 aspect ratio
.Media--flex.Media--ratio2by1
2:1 aspect ratio
.Media--flex.Media--ratio1by1
1:1 aspect ratio 
<figure class="Media [modifier class]">
  <img class="Media-object" src="//dummyimage.com/600/545454/f5f5f5.jpg"/>
</figure>

1.3.1.1Flexible Media Backgrounds #permalink

Flexible backgrounds support the same .Media--ratioXbyY classes. You must also add your own background-image styles (inline rule only an example.)

Example 
<div class="Media Media--flex Media--flexBg Media--ratio16by9" style="background-image: url(//dummyimage.com/600/545454/f5f5f5.jpg);">
  <!-- No content -->
</div>

1.3.1.3Media Constrain (inline) #permalink

This class constrains the image to the parent container instead of stretching it to fill.

Example 
<figure class="Media Media--flex Media--ratio16by9 Media--constrain">
  <img class="Media-object" src="//dummyimage.com/600/545454/f5f5f5.jpg"/>
</figure>
1.3.1.3.1Media Constrain (backgrounds) #permalink

Same as inline except using backgrounds. Browsers that do not support background-size property will ignore this rule.

Compatible in IE9+ and all modern browsers.

Example 
<div class="Media Media--flex Media--flexBackground Media--ratio16by9 Media--constrainBg" style="background-image: url(//dummyimage.com/600/545454/f5f5f5.jpg);">
  <!-- No content -->
</div>
